# 事件日志管理器（LongMemory/EventLogManager）需求文档

## 1. 背景与目标

LongMemory 体系需要一种可长期维护的“事件日志（Event Logs）”能力，用于记录群聊/私聊中发生的、可被后续唤回的事实与进展。

本模块目标：

- 将事件以“原子事件条目”的形式写入存储（向量检索友好）。
- 支持事件的增删改查与相似度检索唤回。
- 支持按会话维度过滤（避免跨群串线）。
- 支持“命中触碰”时间更新（用于上游做降权/清理/排序）。

明确不做：

- 不维护 Active Event List（活跃事件列表由上游系统维护）。
- 不要求 `event_name` 唯一（允许重复；只用于展示/检索辅助）。

## 2. 名词解释

- **事件条目**：一条可检索的事件记录，包含事件详情、相关成员、时间段、会话 ID 等。
- **doc_id**：事件在向量数据库中的记录 ID（Qdrant point id；本实现默认使用 UUID）。
- **相似度**：Qdrant `query_points` 返回的 `score`（越大越相似）。
- **distance**：便捷展示字段，约定为 `1 - similarity`。

## 3. 适用范围

- 适用模块：plugins/GTBot/services/LongMemory/EventLogManager.py
- 存储后端：Qdrant（AsyncQdrantClient）
- 向量生成：VectorGenerator 协议（embed_query/embed_documents）

## 4. 数据结构（对外）

### 4.1 事件输入模型

- `EventLog`：用于新增事件
  - `time_slots: list[TimeSlot]`
  - `details: str`
  - `relevant_members: list[int]`
  - `session_id: int`

### 4.2 事件读取模型

- `EventLogWithId`：用于 `get_by_doc_id` 返回
  - `doc_id: str`
  - `event_name: str`
  - `session_id: int`
  - `relevant_members: list[int]`
  - `time_slots: list[TimeSlot]`
  - `details: str`
  - `status: str`（"open"|"closed"）

### 4.3 事件检索命中

- `EventLogSearchHit`
  - `doc_id, event_name, session_id, relevant_members, details, status`
  - `distance: float`
  - `similarity: float`

## 5. 功能需求（FR）

### FR-1 初始化与建表

- 提供 `QdrantEventLogManager.create(...)`：
  - 若 collection 不存在，则创建。
  - 若未提供 `vector_size`，可通过一次 embedding 探测维度。

验收标准：
- collection 不存在时可自动创建。

### FR-2 新增事件

- 提供 `add_event(event, event_name="", status="open") -> doc_id`。
- `details` 必须非空。
- 写入 payload 字段包含：type/session_id/status/details/relevant_members/time_slots/creation_time/last_updated/last_called_time。

验收标准：
- 返回 doc_id 为字符串。
- 可通过 `get_by_doc_id` 读取到相同的 details/members/session。

### FR-3 获取事件

- 提供 `get_by_doc_id(doc_id, with_vectors=False)`：支持单个或批量。
- doc_id 不存在时抛 `ValueError`。

### FR-4 相似度检索

- 提供 `search_events(query, n_results=5, session_id=None, status="all", relevant_members_any=None, min_similarity=None, order_by="similarity", order="desc", touch_last_called=True)`。
- 支持单条查询与批量查询。
- 支持过滤：session_id、status（open/closed/all）、relevant_members 任意命中。
- 默认命中后更新 `last_called_time`。

验收标准：
- `n_results<=0` 返回空。
- 返回命中项中 similarity/distance 字段存在且为 float。

### FR-5 更新

- 提供 `update_by_doc_id(...) -> int`：支持单个或批量 doc_id。
- 仅更新显式传入字段。
- 若更新语义字段（details/event_name/relevant_members），需要重算向量并 upsert。
- 支持快捷方法：
  - `touch_called_time_by_doc_id`
  - `close_event`

验收标准：
- doc_id 不存在时抛 ValueError。

### FR-6 删除

- 提供 `delete_by_doc_id(doc_id) -> int`。

## 6. 非功能需求（NFR）

- 可靠性：参数校验明确；doc_id 缺失/非法及时报错。
- 可维护性：接口风格与 LongMemory 现有 Qdrant 管理器一致。
- 性能：检索默认限制最大返回 `MAX_N_RESULTS`。

## 7. 不在范围内

- 事件的“合并/月报生成”。
- Active List 的维护、会话生命周期管理。
- 事件名唯一性约束（允许重复）。
