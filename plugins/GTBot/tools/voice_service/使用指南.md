# GTBot 语音插件使用指南

本文说明 `plugins/GTBot/tools/voice_service/` 的用途、配置方式、命令用法、LangChain 工具用法和常见问题。

## 1. 功能概览

本插件当前同时提供 NoneBot 命令和 GTBot LangChain 工具，支持以下能力：

- 切换当前语音模式
- 分别切换“合成模式”和“识别模式”
- 查看当前模式下可用音色
- 设置当前模式下的默认音色
- 合成语音
- 识别语音
- 在阿里云模式下克隆音色
- 在 LangChain 工具层调用语音合成和语音识别

当前支持的模式：

- `QQ API`
- `阿里云 Qwen`
- `阿里云 CosyVoice`

模式边界：

- `QQ API`：支持群聊 QQ AI 语音合成和 QQ 语音识别
- `阿里云 Qwen`：支持 Qwen TTS、Qwen ASR、Qwen 声音复刻
- `阿里云 CosyVoice`：支持 CosyVoice TTS、CosyVoice 声音复刻，不支持语音识别

## 2. 文件位置

插件目录：

```text
plugins/GTBot/tools/voice_service/
```

关键文件：

- `config.json`：实际运行配置
- `config.json.example`：配置样例
- `state.py`：会话状态持久化
- `commands.py`：NoneBot 命令入口
- `tool.py`：LangChain 工具入口
- `providers/qq_api.py`：QQ 侧接口
- `providers/aliyun.py`：阿里云 Qwen 接口
- `providers/cosyvoice.py`：阿里云 CosyVoice 接口
- `使用指南.md`：本文档

运行期数据：

- `./data/state.json`：保存每个会话的模式和音色状态
- `./data/cache/`：缓存下载的语音和转码产物

## 3. 使用前提

### 3.1 基础环境

需要满足以下条件：

- GTBot 已正常运行
- `plugins/GTBot/tools/` 已被 GTBot 插件系统加载
- Python 依赖已安装，包括 `httpx`、`websockets` 和 `dashscope`
- 机器上可执行 `ffmpeg`

### 3.2 QQ 模式前提

QQ 模式通过当前 NoneBot / OneBot 连接上的：

```python
bot.call_api(...)
```

调用 LLOneBot 扩展接口。

因此你需要确保：

- 当前连接的 OneBot 实现是 LLOneBot，或兼容这些扩展接口
- 支持以下接口：
  - `get_ai_characters`
  - `send_group_ai_record`
  - `voice_msg_to_text`
  - `get_record`

注意：

- QQ 音色列表和 QQ AI 语音合成只支持群聊
- 私聊下 QQ 模式无法做 QQ AI 语音合成

### 3.3 阿里云模式前提

阿里云模式需要百炼 API Key。

当前配置分为两套：

- `aliyun.qwen`
- `aliyun.cosyvoice`

默认模型：

- Qwen 识别：`qwen3-asr-flash-realtime`
- Qwen 复刻：`qwen-voice-enrollment`
- Qwen 自定义音色合成：`qwen3-tts-vc-realtime-2026-01-15`
- Qwen 系统音色合成：`qwen3-tts-flash`
- CosyVoice 复刻：`voice-enrollment`
- CosyVoice 合成：`cosyvoice-v3-flash`

注意：

- Qwen 和 CosyVoice 的音色不通用
- Qwen 和 CosyVoice 的 API 格式也不通用
- Qwen 复刻支持本地语音转 `Data URL`
- CosyVoice 复刻要求公网可访问的音频 URL

## 4. 配置说明

配置文件路径：

[config.json](/d:/QQBOT/nonebot/ggz/plugins/GTBot/tools/voice_service/config.json)

示例：

```json
{
  "command_prefix": "语音",
  "scope_mode": "session",
  "permissions": {
    "manage_mode": "admin",
    "set_voice": "admin",
    "synthesize": "all",
    "recognize": "all",
    "clone_voice": "admin"
  },
  "qq_api": {
    "enabled": true,
    "chat_type": 1
  },
  "aliyun": {
    "enabled": true,
    "api_key": "",
    "region": "cn-beijing",
    "asr_model": "qwen3-asr-flash-realtime",
    "timeout_sec": 60.0,
    "qwen": {
      "voice_clone_model": "qwen-voice-enrollment",
      "tts_custom_model": "qwen3-tts-vc-realtime-2026-01-15",
      "tts_builtin_model": "qwen3-tts-flash",
      "language_type": "Chinese",
      "sample_rate": 24000,
      "response_format": "wav"
    },
    "cosyvoice": {
      "enabled": true,
      "voice_clone_model": "voice-enrollment",
      "tts_custom_model": "cosyvoice-v3-flash",
      "tts_builtin_model": "cosyvoice-v3-flash",
      "sample_rate": 24000,
      "response_format": "mp3"
    }
  },
  "audio": {
    "ffmpeg_path": "ffmpeg",
    "max_sample_duration_sec": 60,
    "max_sample_size_mb": 10,
    "target_sample_rate": 16000,
    "target_channels": 1
  },
  "storage": {
    "state_file": "./data/state.json",
    "cache_dir": "./data/cache",
    "temp_ttl_sec": 3600
  }
}
```

### 4.1 关键配置项

`permissions`

- `manage_mode`：谁可以切换模式
- `set_voice`：谁可以查看和设置音色，默认建议为 `admin`
- `synthesize`：谁可以合成语音
- `recognize`：谁可以识别语音
- `clone_voice`：谁可以克隆音色

`qq_api`

- `enabled`：是否启用 QQ 模式
- `chat_type`：传给 LLOneBot 语音接口的会话类型参数

`aliyun`

- `api_key`：百炼 API Key
- `region`：区域
- `asr_model`：Qwen 识别模型
- `timeout_sec`：阿里云接口超时

`aliyun.qwen`

- `voice_clone_model`：Qwen 声音复刻模型
- `tts_custom_model`：Qwen 自定义音色合成模型
- `tts_builtin_model`：Qwen 系统音色合成模型
- `language_type`：Qwen 合成语言类型
- `sample_rate`：Qwen 合成输出采样率
- `response_format`：Qwen 合成输出格式

`aliyun.cosyvoice`

- `enabled`：是否启用 CosyVoice 模式
- `voice_clone_model`：CosyVoice 复刻模型
- `tts_custom_model`：CosyVoice 自定义音色合成模型
- `tts_builtin_model`：CosyVoice 系统音色合成模型
- `sample_rate`：CosyVoice 合成输出采样率
- `response_format`：CosyVoice 合成输出格式

`audio`

- `ffmpeg_path`：`ffmpeg` 可执行文件路径
- `max_sample_duration_sec`：克隆和识别时允许的最大语音时长
- `max_sample_size_mb`：允许的最大文件大小
- `target_sample_rate`：识别和复刻前统一转码采样率
- `target_channels`：统一转码声道数

## 5. 命令说明

### 5.1 同时切换模式

```text
/语音模式 QQ
/语音模式 阿里云Qwen
/语音模式 阿里云CosyVoice
```

行为：

- `QQ`：合成和识别都切到 QQ
- `阿里云Qwen`：合成和识别都切到阿里云 Qwen
- `阿里云CosyVoice`：合成切到 CosyVoice，识别自动切到阿里云 Qwen

### 5.2 单独切换合成模式

```text
/语音合成模式 QQ
/语音合成模式 阿里云Qwen
/语音合成模式 阿里云CosyVoice
```

### 5.3 单独切换识别模式

```text
/语音识别模式 QQ
/语音识别模式 阿里云Qwen
```

说明：

- CosyVoice 不支持语音识别，因此没有 `阿里云CosyVoice` 识别模式

### 5.4 查看音色列表

```text
/语音音色列表
```

作用：

- QQ 合成模式：查看当前群可用的 QQ AI 音色
- 阿里云 Qwen 合成模式：查看 Qwen 系统音色和 Qwen 自定义音色
- 阿里云 CosyVoice 合成模式：查看 CosyVoice 系统音色和 CosyVoice 自定义音色

### 5.5 设置当前音色

```text
/语音设置音色 Cherry
/语音设置音色 longanyang
/语音设置音色 某个QQ音色名
```

作用：

- 为当前会话设置当前“合成模式”下的默认音色

说明：

- QQ、阿里云 Qwen、阿里云 CosyVoice 的当前音色分别存储，互不影响

### 5.6 合成语音

```text
/语音合成 你好，这是测试语音
```

行为：

- QQ 合成模式：走群 AI 语音接口
- 阿里云 Qwen 合成模式：生成音频后以普通 `record` 发回
- 阿里云 CosyVoice 合成模式：生成音频后以普通 `record` 发回

### 5.7 识别语音

操作方式：

1. 回复一条语音消息或音频文件
2. 发送：

```text
/语音识别
```

行为：

- QQ 识别模式：优先走 `voice_msg_to_text`
- 阿里云 Qwen 识别模式：下载语音、转码后走 `qwen3-asr-flash-realtime`

### 5.8 克隆音色

操作方式：

1. 切换到阿里云 Qwen 或阿里云 CosyVoice 合成模式
2. 回复一条语音消息或音频文件
3. 发送：

```text
/语音克隆音色
```

之后会进入分步引导：

1. 输入音色名称
2. 输入目标模型，或发送 `默认`

交互规则：

- 每一步超时时间 30 秒
- 任一步输入 `$` 取消
- 仅发起这次命令的用户可以继续填写

差异：

- Qwen：样本会下载到本地、转成 WAV，再转 `Data URL` 调用 `qwen-voice-enrollment`
- CosyVoice：
  - 如果引用消息中的文本包含链接，会先尝试判断该链接是否为合法音频链接
  - 如果引用的是语音或音频文件，会先上传文件到百炼，再用上传后返回的 URL 调用 `create_voice`
  - 最后通过 `query_voice` 轮询直到返回最终 `voice_id`

## 6. LangChain 工具说明

当前插件已注册两个工具：

- `voice_synthesize`
- `voice_recognize`

### 6.1 `voice_synthesize`

作用：

- 合成并直接发送语音消息

参数：

- `text`：必填，要合成的文本
- `synth_mode`：可选，`qq` / `aliyun_qwen` / `aliyun_cosyvoice`
- `voice_name`：可选，本次临时指定音色名或音色 ID
- `style_text`：可选，用自然语言描述语气、情绪或说法，仅 `aliyun_cosyvoice` 支持，内部会映射到 CosyVoice 的 `instruction`
- `rate`：可选，语速，仅 `aliyun_cosyvoice` 支持
- `pitch`：可选，语调，仅 `aliyun_cosyvoice` 支持
- `volume`：可选，音量，仅 `aliyun_cosyvoice` 支持

注意：

- `style_text`、`rate`、`pitch`、`volume` 传给 QQ 或阿里云 Qwen 时会直接报错
- 这些高级参数当前只在 CosyVoice 模式下生效

### 6.2 `voice_recognize`

作用：

- 识别语音或音频文件消息为文本

参数：

- `message_id`：可选。显式指定目标消息 ID
- `recognize_mode`：可选，`qq` / `aliyun_qwen`

行为：

- 不传 `message_id` 时，默认识别当前回复消息
- 传 `message_id` 时，直接识别指定消息

## 7. 推荐使用流程

### 7.1 QQ 模式

```text
/语音模式 QQ
/语音音色列表
/语音设置音色 <某个QQ群可用音色>
/语音合成 今天天气不错
```

识别：

1. 回复一条语音消息
2. 发送 `/语音识别`

### 7.2 阿里云 Qwen 模式

```text
/语音模式 阿里云Qwen
/语音音色列表
/语音设置音色 Cherry
/语音合成 你好，我是 GTBot
```

克隆并测试：

1. 回复一条语音消息
2. 发送 `/语音克隆音色`
3. 按提示填写音色名称和目标模型
4. 再发送 `/语音合成 这是复刻后的声音`

### 7.3 阿里云 CosyVoice 模式

```text
/语音模式 阿里云CosyVoice
/语音音色列表
/语音设置音色 longanyang
/语音合成 你好，我是 GTBot
```

说明：

- CosyVoice 更适合后续通过工具层传 `rate / pitch / volume`
- CosyVoice 不支持语音识别
- CosyVoice 克隆要求样本消息能解析出公网音频 URL

## 8. 插件内部行为说明

### 8.1 会话状态

插件会保存每个会话的：

- 当前合成模式
- 当前识别模式
- QQ 当前音色
- 阿里云 Qwen 当前音色与 `target_model`
- 阿里云 CosyVoice 当前音色与 `target_model`

状态文件默认在：

[data/state.json](/d:/QQBOT/nonebot/ggz/plugins/GTBot/tools/voice_service/data/state.json)

### 8.2 缓存目录

插件会在缓存目录里保存：

- 下载的原始语音
- 转码后的 WAV / PCM
- 合成得到的临时音频

默认目录：

[data/cache](/d:/QQBOT/nonebot/ggz/plugins/GTBot/tools/voice_service/data/cache)

## 9. 常见问题

### 9.1 `/语音识别` 提示“请先回复一条语音消息”

原因：

- 当前命令侧默认走“回复语音消息或音频文件”

处理：

- 先回复目标语音或音频文件，再发送命令

### 9.2 CosyVoice 为什么不能识别语音

原因：

- 当前插件只实现了 CosyVoice 合成和复刻，没有实现 CosyVoice 识别

处理：

- 识别请切到 `QQ` 或 `阿里云Qwen`

### 9.3 CosyVoice 为什么不能直接克隆本地下载语音

旧行为原因：

- 官方 CosyVoice 复刻接口最终仍然需要 URL

当前行为：

- 如果引用文本里已经有合法音频链接，插件直接使用该链接
- 如果引用的是语音或音频文件，插件会先通过 DashScope `Files.upload` 上传，再使用上传返回的 URL 创建音色

处理：

- 确保环境已安装 `dashscope`
- 文本引用链接时，确保链接确实是可访问的音频 URL

### 9.4 克隆成功但不能合成

常见原因：

- 当前 `tts_custom_model` 和音色保存时的 `target_model` 不一致

处理：

- 把对应模式的 `tts_custom_model` 改回创建这个音色时使用的模型

### 9.5 提示 ffmpeg 失败

处理：

- 确认系统可直接执行 `ffmpeg`
- 或把 `audio.ffmpeg_path` 改成绝对路径

## 10. 安全建议

- 不要把生产环境百炼 API Key 提交到 Git
- 不要把用户语音样本长期保存在缓存目录
- 如果插件要给多人使用，建议保留 `set_voice=admin`
- 如果插件要给多人使用，建议保留 `clone_voice=admin`
