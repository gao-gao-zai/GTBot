# GTBot 配置指南

本文档详细介绍了 GTBot 的配置系统，包括配置文件结构、字段说明以及如何修改配置。

## 1. 配置文件结构概述

GTBot 的配置系统由三个主要部分组成，分别存储在不同的文件中，以便于管理和维护：

1.  **主配置文件 (`config.json`)**: 定义了其他配置文件的路径以及默认使用的配置组。这是配置系统的入口。
2.  **API 配置文件**: 存储所有 LLM (大语言模型) 服务提供商的配置信息，如 API Key、Base URL 和模型参数。
3.  **配置组文件**: 定义了不同的使用场景（配置组），每个配置组指定了使用哪个模型以及相关的提示词（Prompt）。

## 2. 主配置文件 (`config.json`)

默认位置: `plugins/GTBot/config/config.json`

该文件采用 JSON5 格式（支持注释），包含以下字段：

| 字段名                 | 类型  | 说明                                                                                                    | 示例                    |
| :--------------------- | :---- | :------------------------------------------------------------------------------------------------------ | :---------------------- |
| `api_config_path`      | `str` | API 配置文件的路径。可以是绝对路径，也可以是相对于 `ConfigManager.py` 的相对路径。                      | `"./api_config.json"`   |
| `config_groups_path`   | `str` | 配置组文件的路径。可以是绝对路径，也可以是相对于 `ConfigManager.py` 的相对路径。                        | `"./config_group.json"` |
| `default_config_group` | `str` | 默认启动时使用的配置组名称。                                                                            | `"default"`             |
| `prompt_dir_path`      | `str` | 提示词目录的路径。可以是绝对路径，也可以是相对于 `ConfigManager.py` 的相对路径。默认为 `.` (当前目录)。 | `"../../prompts"`       |

**示例内容:**

```json
{
  "api_config_path": "./api_config.json",
  "config_groups_path": "./config_group.json",
  "default_config_group": "default",
  "prompt_dir_path": "../../prompts"
}
```

## 3. API 配置文件

该文件存储了所有服务提供商（Provider）的详细配置。结构是一个字典，Key 为提供商名称（如 `openai`, `anthropic`），Value 为该提供商的配置对象。

### 提供商配置 (Provider)

| 字段名       | 类型   | 说明                                                                   |
| :----------- | :----- | :--------------------------------------------------------------------- |
| `base_url`   | `str`  | API 的基础 URL。例如 OpenAI 为 `https://api.openai.com/v1`。           |
| `api_key`    | `str`  | 该提供商的 API 密钥。                                                  |
| `llm_models` | `dict` | 该提供商下可用的模型列表。Key 为模型简称（自定义），Value 为模型配置。 |

### 模型配置 (LLMModel)

| 字段名             | 类型   | 说明                                           |
| :----------------- | :----- | :--------------------------------------------- |
| `model`            | `str`  | 上游 API 实际使用的模型 ID，如 `gpt-4-turbo`。 |
| `max_input_tokens` | `int`  | 允许输入的最大 Token 数，用于控制成本。        |
| `supports_vision`  | `bool` | 是否支持视觉输入（图片识别）。                 |
| `supports_audio`   | `bool` | 是否支持音频输入。                             |
| `parameters`       | `dict` | 自定义模型参数，如 `temperature`, `top_p` 等。 |

**示例内容:**

```json
{
  "openai": {
    "base_url": "https://api.openai.com/v1",
    "api_key": "sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "llm_models": {
      "gpt4": {
        "model": "gpt-4-turbo",
        "max_input_tokens": 4096,
        "supports_vision": true,
        "supports_audio": false,
        "parameters": {
          "temperature": 0.7
        }
      }
    }
  }
}
```

## 4. 配置组文件

该文件定义了不同的配置组（Configuration Groups）。每个配置组代表一种特定的机器人行为模式或使用场景。结构是一个字典，Key 为配置组名称，Value 为配置组详情。

### 配置组详情 (SingleConfigurationGroup)

每个配置组包含一个 `chat_model` 对象，定义了聊天相关的配置。

| 字段名                                | 类型  | 说明                                                                                                       |
| :------------------------------------ | :---- | :--------------------------------------------------------------------------------------------------------- |
| `model`                               | `str` | 指定使用的模型，格式为 `provider/model_key`。例如 `openai/gpt4` 表示使用 `openai` 提供商下的 `gpt4` 模型。 |
| `maximum_number_of_incoming_messages` | `int` | 最大输入消息数，用于控制上下文长度。                                                                       |
| `behavioral_prompt`                   | `str` | 行为提示词文件的路径。相对路径是基于 `config.json` 中的 `prompt_dir_path` 解析的。                         |
| `character_prompt`                    | `str` | 角色提示词文件的路径。相对路径是基于 `config.json` 中的 `prompt_dir_path` 解析的。                         |

**示例内容:**

```json
{
  "default": {
    "chat_model": {
      "model": "openai/gpt4",
      "maximum_number_of_incoming_messages": 10,
      "behavioral_prompt": "behavioral_default.txt",
      "character_prompt": "character_catgirl.txt"
    }
  }
}
```

## 5. 提示词文件

在配置组中引用的 `behavioral_prompt` 和 `character_prompt` 是纯文本文件。

- **行为提示词**: 定义机器人的行为规范、回复格式等。
- **角色提示词**: 定义机器人的性格、人设、背景故事等。

这两个文件的内容会在运行时被读取并合并，作为最终发送给模型的 System Prompt。

## 6. 配置加载与重载

- **初始化**: 程序启动时，`ConfigManager` 会自动读取 `config.json`，然后根据其中的路径加载 API 配置和配置组。
- **热重载**: 支持在不重启程序的情况下重载配置。
  - 如果修改了配置文件，可以调用 `total_config.reload()` 方法。
  - `reload(keep_current_group=True)`: 重载后尝试保持当前正在使用的配置组。
  - `reload(keep_current_group=False)`: 重载后强制切换回默认配置组。

## 7. 常见问题

- **路径解析规则**:
  - `api_config_path`, `config_groups_path`, `prompt_dir_path`: 如果是相对路径，是相对于 `plugins/GTBot/ConfigManager.py` 文件所在的目录解析的。
  - `behavioral_prompt`, `character_prompt`: 如果是相对路径，是相对于 `prompt_dir_path` 解析的。
- **模型名称限制**: 提供商名称（Provider Name）和模型简称（Model Key）中**不允许包含 `/` 字符**，因为 `/` 被用作分隔符（`provider/model`）。
- **模型引用**: 在配置组中指定模型时，必须使用 `provider/model_key` 的格式，且 `provider` 必须在 API 配置文件中存在，`model_key` 必须在该提供商的 `llm_models` 中存在。
