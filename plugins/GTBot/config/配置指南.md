# GTBot 配置指南

本文档说明 GTBot 的主要配置文件结构、关键字段以及常见配置方式。

## 1. 配置文件结构

GTBot 的配置主要分为三部分：

1. `config.json`
   指定其他配置文件路径，以及默认使用的配置组。
2. `api_config.json`
   定义模型提供商、API 地址、密钥、模型列表和模型参数。
3. `config_group.json`
   定义不同场景下使用哪个模型、提示词和会话相关参数。

## 2. 主配置文件 `config.json`

默认位置：`plugins/GTBot/config/config.json`

示例：

```json
{
  "api_config_path": "./api_config.json",
  "config_groups_path": "./config_group.json",
  "default_config_group": "default",
  "prompt_dir_path": "../../prompts",
  "data_dir_path": "./data",
  "user_cache_update_interval_sec": 3600,
  "user_cache_expire_sec": 604800
}
```

字段说明：

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `api_config_path` | `str` | API 配置文件路径。相对路径相对于 `ConfigManager.py` 解析。 |
| `config_groups_path` | `str` | 配置组文件路径。 |
| `default_config_group` | `str` | 默认启用的配置组名称。 |
| `prompt_dir_path` | `str` | 提示词目录路径。 |
| `data_dir_path` | `str` | 数据目录路径，不存在时会自动创建。 |
| `user_cache_update_interval_sec` | `int` | 用户缓存刷新间隔。 |
| `user_cache_expire_sec` | `int` | 用户缓存过期时间。 |

## 3. API 配置文件 `api_config.json`

该文件保存所有 Provider 的配置。顶层 key 是你自定义的提供商名称，例如 `openai`、`anthropic_main`、`aliyun_qwen`。

### 3.1 Provider 配置

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `provider_type` | `str` | 提供商类型。可选：`openai_compatible`、`openai_responses`、`anthropic`、`gemini`、`dashscope`。默认 `openai_compatible`。 |
| `process_tool_call_deltas` | `bool` | GTBot 内部开关，可写在 provider 根级作为默认值，控制是否处理流式工具调用增量。 |
| `base_url` | `str` | API 基础 URL。部分 provider type 必填。 |
| `api_key` | `str` | API 密钥。 |
| `llm_models` | `dict` | 当前 provider 下可用的模型列表。key 为模型别名，value 为模型配置。 |

### 3.2 模型配置

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `model` | `str` | 上游实际模型 ID。 |
| `max_input_tokens` | `int` | 模型允许的最大输入 token 数。 |
| `supports_vision` | `bool` | 是否支持视觉输入。 |
| `supports_audio` | `bool` | 是否支持音频输入。 |
| `parameters` | `dict` | 模型参数，例如 `temperature`、`top_p`、`stream` 等；也支持 GTBot 内部参数 `process_tool_call_deltas`。 |

### 3.3 示例

```json
{
  "openai_main": {
    "provider_type": "openai_compatible",
    "base_url": "https://api.openai.com/v1",
    "api_key": "sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "llm_models": {
      "gpt4": {
        "model": "gpt-4-turbo",
        "max_input_tokens": 4096,
        "supports_vision": true,
        "supports_audio": false,
        "parameters": {
          "temperature": 0.7,
          "process_tool_call_deltas": true
        }
      }
    }
  }
}
```

### 3.4 `provider_type` 说明

- `openai_compatible`
  使用 OpenAI 兼容聊天接口，适用于大多数 `/v1/chat/completions` 风格服务。
- `openai_responses`
  使用 OpenAI Responses API。
- `anthropic`
  使用 Anthropic 原生 LangChain 适配器。
- `gemini`
  使用 Google Gemini 原生 LangChain 适配器。
- `dashscope`
  使用阿里云百炼 / Tongyi 原生 LangChain 适配器。

注意：

- 配置组中的 `provider/model_key` 里，`provider` 只是你自定义的名称。
- 真正决定底层客户端类型的是该 provider 对应的 `provider_type`。
- 如果使用 `anthropic`、`gemini`、`dashscope`，需要安装对应依赖。

## 4. 配置组文件 `config_group.json`

配置组用于定义不同场景下的聊天模型和提示词。

示例：

```json
{
  "default": {
    "chat_model": {
      "model": "openai_main/gpt4",
      "maximum_number_of_incoming_messages": 10,
      "behavioral_prompt": "behavioral_default.txt",
      "character_prompt": "character_catgirl.txt"
    },
    "message_format_placeholder": "[[$time_M]-[$time_d] [$time_h]:[$time_m]:[$time_s]] [$user_name]([$user_id]):[$message]"
  }
}
```

字段说明：

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `model` | `str` | 使用的模型，格式必须为 `provider/model_key`。 |
| `maximum_number_of_incoming_messages` | `int` | 会话上下文中保留的最大消息数。 |
| `behavioral_prompt` | `str` | 行为提示词文件路径，相对路径基于 `prompt_dir_path`。 |
| `character_prompt` | `str` | 角色提示词文件路径，相对路径基于 `prompt_dir_path`。 |
| `message_format_placeholder` | `str` | 消息格式化模板。 |

## 5. `message_format_placeholder`

该字段控制历史消息被格式化后送入模型时的文本样式。

常见占位符：

| 占位符 | 说明 |
| :--- | :--- |
| `[$time_Y]` | 年 |
| `[$time_M]` | 月 |
| `[$time_d]` | 日 |
| `[$time_h]` | 时 |
| `[$time_m]` | 分 |
| `[$time_s]` | 秒 |
| `[$user_id]` | 用户 ID |
| `[$user_name]` | 用户昵称 |
| `[$group_id]` | 群号 |
| `[$message_id]` | 消息 ID |
| `[$message]` | 消息内容 |

示例：

```text
[[$time_M]-[$time_d] [$time_h]:[$time_m]:[$time_s]] [$user_name]([$user_id]):[$message]
```

## 6. 配置加载与重载

- 启动时会先读取 `config.json`，再加载 `api_config.json` 和 `config_group.json`。
- 支持调用 `total_config.reload()` 热重载配置。
- `reload(keep_current_group=True)` 会尽量保留当前配置组。
- `reload(keep_current_group=False)` 会切回默认配置组。

## 7. 常见注意事项

- Provider 名称和模型别名都不能包含 `/`，因为 `/` 被用作 `provider/model_key` 分隔符。
- `provider_type` 为 `openai_compatible`、`openai_responses`、`anthropic` 时通常要求 `base_url` 非空。
- 并不是所有模型都支持工具调用、视觉输入或流式输出，是否可用还取决于上游模型本身。
- `parameters` 中的值会透传给对应 provider 的 LangChain 客户端，建议只填写该 provider 支持的参数。
- `process_tool_call_deltas` 是 GTBot 内部开关，默认 `true`。可写在 provider 根级作为该 provider 的默认值，也可写在某个模型的 `parameters` 里单独覆盖。
- 关闭 `process_tool_call_deltas` 后会停用流式工具调用增量处理，也不会再对阿里 DashScope 的 OpenAI 兼容接口自动降级为非流式。

## 8. 用户缓存配置

- `user_cache_update_interval_sec`
  控制缓存多久后需要刷新。
- `user_cache_expire_sec`
  控制长时间未访问的缓存多久被清理。

缓存数据保存在 `data/data.db` 中，依赖 `nonebot-plugin-apscheduler` 进行定时刷新和清理。
