
# 代码改造计划（插件系统落地）

> 目标：用 **插件化** 替换现有“外置工具”机制，支持 `register(registry)` 注册  
> - tools  
> - `AgentMiddleware`  
> - callbacks（通过 `RunnableConfig.callbacks` / `.with_config(...)` 注入）  
> 同时提供 `raw_messages: list[GroupMessage]` 供插件**读取分析**；**上下文改写通过中间件实现**（不做 transforms 机制）。

---

## 0. 总原则（保证最小侵入/可回滚）
- **最小行为变化**：先保证“现有内置 tools + LongMemory tools + DirectAssistantOutputMiddleware”行为不变。
- **渐进替换**：先引入插件系统并接入，但保留 [ToolLoader](cci:2://file:///mnt/d/QQBOT/nonebot/ggz/plugins/GTBot/services/tool_loader.py:13:0-124:28) 一段时间作为 fallback（可通过配置/开关控制）。
- **每次请求构建 agent**：沿用你现在 [create_group_chat_agent()](cci:1://file:///mnt/d/QQBOT/nonebot/ggz/plugins/GTBot/Chat.py:505:0-594:16) 的调用方式（无需做全局 agent 热替换复杂逻辑）。

## 1. 阶段一：引入插件系统核心（不改业务调用链）
### 1.1 新增目录/文件（新增不破坏现有逻辑）
建议新增包目录（路径可调整，但建议放 services 下）：
- `plugins/GTBot/services/plugin_system/__init__.py`
- `plugins/GTBot/services/plugin_system/types.py`
  - `PluginContext`（包含 `raw_messages`、`runtime_context` 可选、以及 `extra` 可选）

    - `add_callback`
- `plugins/GTBot/services/plugin_system/loader.py`
  - `PluginLoader`：从目录加载模块，调用 `register(registry)`
  - 支持 `reload()`（`importlib.reload`）
  - 错误隔离：单插件异常只记录日志，不影响整体
- `plugins/GTBot/services/plugin_system/runtime.py`
  - ContextVar：`set_current_plugin_context(ctx)` / `get_current_plugin_context()`

### 1.2 插件目录约定（先复用已有 `tools_dir`，后续再拆）

- 插件模块必须实现 `register(registry)`，否则视为“旧工具模块”（见阶段四兼容）

**交付物**

## 5. 风险点与回滚方案
### 5.1 风险点
- **LangChain 版本差异**：`.with_config`/callbacks 透传可能在你的具体版本表现不同
  - 已设计 ContextVar 兜底，不阻断主流程
- **热重载副作用**：reload 后模块状态残留（全局变量、单例）
  - 需要约定插件尽量无全局副作用；或在 reload 时重新实例化 handler/tool
- **流式路径差异**：你有 [_invoke_agent_with_streaming_to_queue](cci:1://file:///mnt/d/QQBOT/nonebot/ggz/plugins/GTBot/Chat.py:707:0-898:23) 这种自定义流式
  - 必须确保 ContextVar 覆盖该协程的生命周期

### 5.2 回滚点
- 保留配置开关：`enable_plugin_system: bool`
  - 关闭时仍走 [get_current_tools()](cci:1://file:///mnt/d/QQBOT/nonebot/ggz/plugins/GTBot/LLM_Tools.py:71:0-72:23) + ToolLoader 老逻辑
- 或者仅在 [create_group_chat_agent](cci:1://file:///mnt/d/QQBOT/nonebot/ggz/plugins/GTBot/Chat.py:505:0-594:16) 里当 plugin_manager 失败时 fallback 到旧 tools（记录 error）

## 6. 验收 checklist（建议你每阶段完成后自测）
- **基础**：不启用插件时，机器人行为与当前一致
- **可见性**：任意 tool/middleware 能读取 `runtime.context.raw_messages`
- **注入**：一个示例插件能成功注册
  - 1 个 tool
  - 1 个 AgentMiddleware
  - 1 个 callback（至少能触发一次）
- **重载**：`重载工具` 命令执行后，下一次对话能用到新插件逻辑
- **流式**：流式开启时，中间件/ContextVar 不丢失上下文
