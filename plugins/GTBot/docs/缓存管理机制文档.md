# 缓存管理机制文档

## 概述

`UserCacheManager` 是 GTBot 中核心的缓存管理系统，用于管理 QQ 群信息、群成员信息、陌生人信息等用户相关数据的缓存生命周期。该系统采用分层缓存策略，结合数据库持久化，提供高效、可靠的数据缓存服务。

---

## 核心特性

### 1. **多层次缓存架构**

缓存系统支持三种主要数据类型的管理：

| 缓存类型       | 数据源                  | 主键                  | 用途                             |
| -------------- | ----------------------- | --------------------- | -------------------------------- |
| **群信息缓存** | `get_group_info`        | `group_id`            | 存储群基本信息（名称、成员数等） |
| **群成员缓存** | `get_group_member_info` | `(group_id, user_id)` | 存储成员详细信息（昵称、头衔等） |
| **陌生人缓存** | `get_stranger_info`     | `user_id`             | 存储非群成员用户信息（昵称等）   |

### 2. **智能刷新机制**

缓存采用时间戳驱动的自动刷新策略：

```python
# 配置参数
user_cache_update_interval_sec = 3600    # 缓存刷新间隔（秒），默认 1 小时
user_cache_expire_sec = 86400            # 缓存过期间隔（秒），默认 24 小时
```

**刷新规则**：

- 首次访问时，自动从 QQ API 获取数据
- 后续访问若距离上次更新时间超过 `update_interval`，自动刷新
- 可通过 `force_refresh=True` 强制立即刷新

**过期规则**：

- 距离上次访问时间超过 `expire_interval` 的记录会被清理
- 定期调用 `cleanup_expired_entries()` 执行清理

### 3. **数据库持久化**

所有缓存数据都存储在 SQLite 数据库中，支持：

- **原子性**：使用 SQLAlchemy ORM 事务管理
- **索引优化**：在 `last_update_time` 和 `last_access_time` 上建立索引
- **异步操作**：完全异步数据库访问，无阻塞 I/O
- **故障恢复**：程序重启后自动从数据库恢复缓存

### 4. **错误处理与降级**

- 所有 API 调用异常都会被捕获并记录
- 获取数据失败时返回默认值（ID 号）而非抛出异常
- 支持快速降级，保证业务连续性

## API 接口

### 基础查询方法

#### 1. `get_group_info(bot, group_id, force_refresh=False)`

获取群信息，返回 `GroupInfo` 对象。

```python
from plugins.GTBot import get_user_cache_manager

async def handle_message(bot: Bot, event: GroupMessageEvent):
    cache_manager = await get_user_cache_manager()

    # 获取群信息（使用缓存）
    group_info = await cache_manager.get_group_info(bot, event.group_id)

    # 强制刷新最新信息
    group_info = await cache_manager.get_group_info(
        bot, event.group_id, force_refresh=True
    )
```

**GroupInfo 对象属性**：

```python
group_info.group_id          # 群号
group_info.group_name        # 群名称
group_info.group_memo        # 群备注
group_info.member_count      # 成员数量
group_info.max_member_count  # 最大成员数量
group_info.is_shutup_all     # 是否全体禁言（属性）
group_info.bot_join_time     # Bot加入时间（属性）
```

#### 2. `get_group_member_info(bot, group_id, user_id, force_refresh=False)`

获取群成员信息，返回 `GroupMemberInfo` 对象。

```python
async def handle_message(bot: Bot, event: GroupMessageEvent):
    cache_manager = await get_user_cache_manager()

    member_info = await cache_manager.get_group_member_info(
        bot, event.group_id, event.user_id
    )
```

**GroupMemberInfo 对象属性**：

```python
member_info.group_id          # 群号
member_info.user_id           # 用户 QQ
member_info.card              # 群名片
member_info.nickname          # 昵称
member_info.role              # 角色 (owner/admin/member)
member_info.title             # 头衔
member_info.display_name      # 显示名称（属性，优先名片）
member_info.shut_up_timestamp # 禁言时间戳
```

#### 3. `get_stranger_info(bot, user_id, force_refresh=False)`

获取陌生人信息，返回 `StrangerInfo` 对象。

```python
async def handle_message(bot: Bot, event: PrivateMessageEvent):
    cache_manager = await get_user_cache_manager()

    user_info = await cache_manager.get_stranger_info(bot, event.user_id)
```

**StrangerInfo 对象属性**：

```python
user_info.user_id       # 用户 QQ
user_info.nickname      # 昵称
user_info.age           # 年龄
user_info.sex           # 性别
user_info.qid           # QID
user_info.level         # 等级
```

### 快捷获取名称方法

#### 1. `get_group_name(bot, group_id, force_refresh=False)`

快速获取群名称，失败时返回群号。

```python
group_name = await cache_manager.get_group_name(bot, event.group_id)
# 返回: "某某群" 或 "123456789"
```

#### 2. `get_user_name(bot, user_id, force_refresh=False)`

快速获取用户昵称，失败时返回用户 ID。

```python
user_name = await cache_manager.get_user_name(bot, event.user_id)
# 返回: "某某用户" 或 "987654321"
```

#### 3. `get_group_member_name(bot, group_id, user_id, force_refresh=False)`

快速获取群成员名称（优先群名片），失败时返回用户 ID。

```python
member_name = await cache_manager.get_group_member_name(
    bot, event.group_id, event.user_id
)
# 返回: "群名片" 或 "昵称" 或 "987654321"
```

### 维护方法

#### 1. `set_bot_self_id(bot)`

设置 bot 自身信息并将其加入缓存。**建议在 bot 启动时调用**。

```python
@bot.on_startup
async def on_startup(bot: Bot):
    cache_manager = await get_user_cache_manager()
    await cache_manager.set_bot_self_id(bot)
```

#### 2. `refresh_due_entries(bot)`

批量刷新所有超时的缓存记录。建议定期调用（如每小时一次）。

```python
# 在定时任务中使用
@scheduler.scheduled_job("interval", hours=1)
async def refresh_cache():
    cache_manager = await get_user_cache_manager()
    await cache_manager.refresh_due_entries(bot)
```

#### 3. `cleanup_expired_entries()`

删除长期未访问的过期缓存记录。建议定期调用（如每天一次）。

```python
# 在定时任务中使用
@scheduler.scheduled_job("cron", hour=3, minute=0)
async def cleanup_cache():
    cache_manager = await get_user_cache_manager()
    await cache_manager.cleanup_expired_entries()
```

#### 4. `ensure_ready()`

确保缓存管理器初始化完成（创建数据库表等）。

```python
cache_manager = await get_user_cache_manager()  # 自动调用 ensure_ready
```

## 使用示例

### 示例 1：处理群消息并获取发送者信息

```python
from nonebot.adapters.onebot.v11 import Bot, GroupMessageEvent
from plugins.GTBot import get_user_cache_manager

async def handle_group_message(bot: Bot, event: GroupMessageEvent):
    """处理群消息"""
    cache_manager = await get_user_cache_manager()

    # 快速获取群名和发送者名称
    group_name = await cache_manager.get_group_name(bot, event.group_id)
    member_name = await cache_manager.get_group_member_name(
        bot, event.group_id, event.user_id
    )

    logger.info(f"[{group_name}] {member_name}: {event.get_message()}")
```

### 示例 2：检查群状态

```python
async def check_group_status(bot: Bot, group_id: int):
    """检查群的禁言状态"""
    cache_manager = await get_user_cache_manager()

    group_info = await cache_manager.get_group_info(bot, group_id)

    if group_info.is_shutup_all:
        logger.warning(f"群 {group_info.group_name} 处于全体禁言状态")

    logger.info(f"群成员数: {group_info.member_count}/{group_info.max_member_count}")
```

### 示例 3：获取群成员权限

```python
async def check_member_permission(bot: Bot, group_id: int, user_id: int) -> bool:
    """检查成员是否为管理员"""
    cache_manager = await get_user_cache_manager()

    member_info = await cache_manager.get_group_member_info(
        bot, group_id, user_id
    )

    return member_info.role in ["admin", "owner"]
```

### 示例 4：定期维护缓存

```python
from apscheduler.schedulers.asyncio import AsyncIOScheduler

scheduler = AsyncIOScheduler()

@scheduler.scheduled_job("interval", hours=1)
async def refresh_cache(bot: Bot):
    """每小时刷新一次缓存"""
    cache_manager = await get_user_cache_manager()
    await cache_manager.refresh_due_entries(bot)

@scheduler.scheduled_job("cron", hour=3, minute=0)
async def cleanup_cache():
    """每天凌晨 3 点清理过期缓存"""
    cache_manager = await get_user_cache_manager()
    await cache_manager.cleanup_expired_entries()
```

## 配置说明

### 配置文件位置

缓存配置在 `config.json` 或 `config_group.toml` 中：

```json
{
  "user_cache_update_interval_sec": 3600, // 缓存刷新间隔（秒）
  "user_cache_expire_sec": 86400 // 缓存过期间隔（秒）
}
```

或 TOML 格式：

```toml
[cache]
user_cache_update_interval_sec = 3600
user_cache_expire_sec = 86400
```

### 参数说明

| 参数                             | 默认值 | 说明                       |
| -------------------------------- | ------ | -------------------------- |
| `user_cache_update_interval_sec` | 3600   | 缓存多久后需要刷新（秒）   |
| `user_cache_expire_sec`          | 86400  | 缓存多久后过期并删除（秒） |

**调优建议**：

- 频繁变更的群信息（如群名称）：设置较小的 `update_interval`
- 稳定的信息：设置较大的 `update_interval` 以减轻 API 压力
- 一般情况下不需要修改 `expire_sec`

## 性能特性

### 缓存命中率

- **首次访问**：从 QQ API 获取（100% API 调用）
- **缓存有效期内**：直接返回数据库缓存（0% API 调用）
- **缓存过期后**：自动刷新，后续仍使用缓存

在配置合理的情况下，缓存命中率通常在 **90%+**，大幅降低 API 调用频率。

### 数据库性能

- **数据库引擎**：SQLite + aiosqlite（异步）
- **存储位置**：`{data_dir}/data.db`
- **表结构**：三个主表 + 索引优化
- **查询速度**：单条查询 < 1ms

### 内存占用

- **运行时内存**：< 10MB（仅缓存管理器）
- **数据库文件**：取决于缓存记录数，通常 < 100MB

## 故障排查

### 问题 1：缓存长期未更新

**症状**：获取的信息总是旧数据

**原因**：可能是 `update_interval` 设置过大，或 `refresh_due_entries()` 未被调用

**解决**：

1. 检查配置中 `user_cache_update_interval_sec` 的值
2. 确保定时任务已启动
3. 手动调用 `force_refresh=True` 强制刷新

```python
# 强制刷新
info = await cache_manager.get_group_info(bot, group_id, force_refresh=True)
```

### 问题 2：数据库文件过大

**症状**：`data.db` 文件占用空间很大

**原因**：缓存记录未被清理（过期数据仍存在）

**解决**：

1. 确保 `cleanup_expired_entries()` 定期被调用
2. 手动调用清理函数

```python
cache_manager = await get_user_cache_manager()
await cache_manager.cleanup_expired_entries()
```

### 问题 3：API 调用错误

**症状**：日志中出现 `获取群名称失败` 之类的错误

**原因**：QQ API 返回错误（权限不足、群不存在等）

**解决**：

1. 检查 bot 是否有相应权限
2. 查看具体错误日志
3. 考虑设置较大的 `update_interval` 减少重试

## 最佳实践

### 1. **启动时初始化**

```python
@bot.on_startup
async def init_cache(bot: Bot):
    cache_manager = await get_user_cache_manager()
    await cache_manager.set_bot_self_id(bot)
    logger.info("缓存管理器已初始化")
```

### 2. **使用快捷方法**

尽量使用 `get_group_name()` 等快捷方法而非直接调用 API：

```python
# ✅ 推荐
name = await cache_manager.get_group_name(bot, group_id)

# ❌ 避免
info = await bot.get_group_info(group_id=group_id)  # 直接调用 API
```

### 3. **定期维护**

```python
# 每小时刷新
@scheduler.scheduled_job("interval", hours=1)
async def refresh():
    await cache_manager.refresh_due_entries(bot)

# 每天清理
@scheduler.scheduled_job("cron", hour=3)
async def cleanup():
    await cache_manager.cleanup_expired_entries()
```

### 4. **合理使用 force_refresh**

```python
# 只在必要时强制刷新
# ✅ 好的做法
if critical_operation:
    info = await cache_manager.get_group_info(
        bot, group_id, force_refresh=True
    )

# ❌ 避免频繁使用
for group_id in many_groups:
    # 不要在循环中频繁强制刷新
    info = await cache_manager.get_group_info(
        bot, group_id, force_refresh=True
    )
```

### 5. **异常处理**

所有快捷方法已包含异常处理，返回默认值：

```python
# 快捷方法总是返回字符串，不会抛异常
name = await cache_manager.get_group_name(bot, invalid_group_id)
# 返回: "invalid_group_id" 的字符串形式
```
