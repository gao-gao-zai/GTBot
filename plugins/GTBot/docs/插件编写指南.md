# GTBot 插件编写指南

本文档面向 GTBot 的 **新插件系统**（`services/plugin_system`），用于指导你如何编写、加载、调试插件。

## 1. 插件放在哪里？如何配置？

- **配置项**：`plugin_dir`
- **默认目录**：通常为 `plugins/GTBot/tools/`（以你的配置为准）
- **加载方式**：宿主启动时会扫描并导入 `plugin_dir` 下的插件入口，支持两种形态：
  - 单文件插件：`plugin_dir/*.py`（排除以下划线开头与 `__init__.py`）
  - 包插件：`plugin_dir/<plugin_name>/__init__.py`

建议：

- 插件文件名使用清晰语义，例如 `tavily_search_plugin.py`。
- `plugin_dir` 下保留 `__init__.py` 作为 Python 包，有利于 IDE/类型检查与绝对导入稳定性。

### 1.1 包插件（Python 包）示例

当插件逻辑较复杂（需要拆分多个文件）时，建议使用包插件：

```text
tools/
  __init__.py
  my_plugin/
    __init__.py  # 必须提供 register(registry)
    core.py
    utils.py
```

约束：

- 包插件的入口固定为 `my_plugin/__init__.py`，宿主会导入 `tools.my_plugin` 并调用其 `register(registry)`。
- 若包目录中不存在 `__init__.py`，则不会被当作插件加载。

## 2. 插件的最小形态：`register(registry)`

每个插件模块必须暴露一个顶层函数：

```python
def register(registry) -> None:
    ...
```

约束：

- 宿主只识别 `register(registry)` 这一种插件接口。
- 模块 **没有** `register` 会被忽略（不会加载任何能力）。

## 3. 你能注册什么？（PluginRegistry API）

`registry` 为 `PluginRegistry` 实例，支持注册以下能力：

- `registry.add_tool(tool, *, priority=0, enabled=None)`
- `registry.add_tool_factory(factory, *, priority=0, enabled=None)`
- `registry.add_agent_middleware(middleware, *, priority=0, enabled=None)`
- `registry.add_callback(callback, *, priority=0, enabled=None)`

### 3.1 priority 合并顺序

- `priority` 越小越靠前（越先合并到最终 `PluginBundle`）。
- 建议：除非你确实需要抢先/兜底逻辑，否则保持默认 `priority=0`。

### 3.2 enabled 按上下文启用

`enabled` 是一个函数：`(ctx: PluginContext) -> bool`。

典型用法：

- 只在某些群启用
- 只在特定用户启用
- 根据 `raw_messages` 内容动态启用

## 4. 插件上下文：PluginContext 与 raw_messages

宿主在每次对话调用时会构造一个 `PluginContext`，核心字段：

- `raw_messages: list[Any]`
  - 约定为 `list[plugins.GTBot.model.GroupMessage]`
  - 用于插件读取“本次对话原始消息（领域模型）”
- `runtime_context: Any | None`
  - 宿主运行期上下文（通常为 `GroupChatContext`）
- `extra: dict[str, Any]`
  - 插件之间共享的临时数据容器（同一次调用内）

### 4.1 在工具/中间件/回调里获取 PluginContext

插件系统通过 `ContextVar` 传播上下文，你可以在任意插件代码中读取：

```python
from plugins.GTBot.services.plugin_system.runtime import get_current_plugin_context

ctx = get_current_plugin_context()
if ctx is not None:
    raw_messages = ctx.raw_messages
```

注意：

- 在插件代码运行于“宿主已进入插件上下文作用域”时，`get_current_plugin_context()` 才会返回非空。
- 在模块 import 顶层（加载阶段）不要依赖该上下文。

## 5. 注册 Tool（工具）

GTBot 侧工具通常使用 LangChain 的 `@tool` 装饰器。

### 5.1 最简单的 tool

```python
from langchain.tools import tool

@tool("my_echo")
def my_echo(text: str) -> str:
    """回显输入。"""
    return text


def register(registry) -> None:  # noqa: ANN001
    registry.add_tool(my_echo)
```

### 5.2 需要访问运行期上下文（ToolRuntime.context）

如果你需要访问宿主的 `GroupChatContext`，可以声明带 `ToolRuntime` 的参数（示例参考 `demo_plugin.py`）：

```python
from langchain.tools import ToolRuntime, tool
from plugins.GTBot.GroupChatContext import GroupChatContext

@tool("my_tool")
def my_tool(runtime: ToolRuntime[GroupChatContext]) -> str:
    ctx = runtime.context
    raw_messages = getattr(ctx, "raw_messages", [])
    return f"raw_messages={len(raw_messages)}"
```

建议：

- 对 `runtime.context` 属性使用 `getattr` 兼容字段缺失，避免插件把宿主打挂。

## 6. 注册 Tool Factory（动态创建工具）

当你的工具需要根据上下文动态启用/注入数据时，建议使用 `add_tool_factory`。

```python
from langchain.tools import tool


def make_tools(ctx):  # noqa: ANN001
    @tool("my_last_message")
    def my_last_message() -> str:
        if not ctx.raw_messages:
            return "(empty)"
        last = ctx.raw_messages[-1]
        return str(getattr(last, "content", ""))

    return [my_last_message]


def register(registry) -> None:  # noqa: ANN001
    registry.add_tool_factory(make_tools)
```

说明：

- `factory` 的签名为 `(PluginContext) -> list[tool]`。
- factory 里闭包捕获 `ctx` 是允许的；但请注意不要把不可序列化对象扩散到全局。

## 7. 注册 Agent Middleware（中间件）

中间件用于在 Agent 执行流程中注入行为（例如：审计、限流、改写输出等）。

示例参考 `plugins/GTBot/tools/demo_plugin.py`：

- 继承 `langchain.agents.middleware.AgentMiddleware`
- 在 `before_model` / `after_model` 等钩子里读取 `runtime.context` 或 `get_current_plugin_context()`

建议：

- 中间件应 **小心异常**：尽量不要抛出未处理异常。
- 如需写共享状态，优先写入 `PluginContext.extra`，避免篡改宿主上下文字段。

## 8. 注册 Callback（回调）

用于接入 LangChain callbacks（链路追踪、事件记录等）。

```python
from typing import Any
from langchain_core.callbacks import BaseCallbackHandler
from plugins.GTBot.services.plugin_system.runtime import get_current_plugin_context


class MyCallback(BaseCallbackHandler):
    def on_chain_start(self, serialized: dict[str, Any], inputs: dict[str, Any], **kwargs: Any) -> None:
        ctx = get_current_plugin_context()
        if ctx is not None:
            ctx.extra["my_callback_seen_raw_messages"] = len(ctx.raw_messages)


def register(registry) -> None:  # noqa: ANN001
    registry.add_callback(MyCallback())
```

## 9. 安全与配置最佳实践

- **不要硬编码 API Key**：
  - 优先从环境变量读取，例如 `os.getenv("TAVILY_API_KEY")`
  - 或接入你现有的配置体系（但避免把密钥提交到仓库）
- **对外部请求做超时**：例如 `httpx.AsyncClient(timeout=20.0)`
- **返回给模型的内容要做裁剪**：
  - 避免返回超大字段（会浪费 tokens、变慢、也容易触发模型上下文限制）

## 10. 调试与重载

### 10.1 运行期重载

项目中提供了重载命令（见 `plugins/GTBot/LLM_Tools.py`）：

- 命令：`重载工具`（别名 `reload_tools`）
- 作用：触发插件目录重新扫描与模块热重载

### 10.2 常见问题排查

- 插件没生效：
  - 确认插件文件位于 `plugin_dir`
  - 确认模块存在 `register(registry)`
  - 触发一次 `重载工具`
- `get_current_plugin_context()` 为 None：
  - 说明当前代码不在宿主设置的插件上下文作用域内；不要在 import 顶层调用它。

## 11. 一个完整插件模板（可复制）

```python
from __future__ import annotations

import os
from typing import Any

from langchain.tools import tool

from plugins.GTBot.services.plugin_system.runtime import get_current_plugin_context


@tool("my_plugin_tool")
def my_plugin_tool(query: str) -> str:
    """示例工具：读取 PluginContext 并返回一些信息。"""
    ctx = get_current_plugin_context()
    raw_count = len(ctx.raw_messages) if ctx is not None else 0
    env_flag = os.getenv("MY_PLUGIN_FLAG", "")
    return f"query={query} raw_messages={raw_count} MY_PLUGIN_FLAG={env_flag}"


def register(registry) -> None:  # noqa: ANN001
    registry.add_tool(my_plugin_tool)
```
