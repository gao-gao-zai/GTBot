# 被拒绝时的表情贴回应功能

## 功能说明

当响应被拒绝（锁已满）时，系统可以自动为用户的消息添加一个表情贴回应，用来表示"系统繁忙"或"已收到但无法立即处理"。

---

## 配置方法

在 `config_group.json` 中添加 `rejection_emoji_id` 参数：

```json
{
  "default": {
    "chat_model": {
      "model": "h/deepseek-v3.1-terminus",
      "maximum_number_of_incoming_messages": 30,
      "max_message_length": 500,
      "behavioral_prompt": "行为提示词/默认.txt",
      "character_prompt": "角色提示词/天天.md",
      "max_concurrent_responses_per_group": 1,
      "max_total_concurrent_responses": 5,
      "rejection_emoji_id": -1
    },
    "message_format_placeholder": "[[$time_M]-[$time_d] [$time_h]:[$time_m]:[$time_s]] [$user_name]([$user_id], [$message_id]):[$message]"
  }
}
```

---

## 参数说明

| 参数                 | 类型 | 默认值 | 说明                                                     |
| -------------------- | ---- | ------ | -------------------------------------------------------- |
| `rejection_emoji_id` | int  | -1     | QQ 表情 ID。-1 表示不开启表情回应，其他值为具体的表情 ID |

---

## 常用表情 ID

| 表情         | ID  | 说明         |
| ------------ | --- | ------------ |
| 👍（点赞）   | 6   | 表示已收到   |
| 💔（难过）   | 34  | 表示系统繁忙 |
| 😂（大笑）   | 102 | 表示无法响应 |
| 😅（冷汗）   | 106 | 表示忙碌中   |
| 😱（惊吓）   | 128 | 表示异常     |
| 😴（打哈欠） | 12  | 表示累了     |

**完整的表情 ID 列表需要根据你的 QQ 版本查询。**

## 使用示例

### 示例 1：不开启表情回应

```json
"rejection_emoji_id": -1
```

被拒绝时：无任何表情回应，只记录日志。

### 示例 2：用点赞表示已收到

```json
"rejection_emoji_id": 6
```

被拒绝时：用 👍 表情回应用户的消息。

### 示例 3：用冷汗表示忙碌中

```json
"rejection_emoji_id": 106
```

被拒绝时：用 😅 表情回应用户的消息。

## 工作流程

```
用户发送 @机器人 消息
    ↓
系统检查响应锁
    ├─ 锁可用 → 处理请求
    └─ 锁满 → 拒绝请求
           ├─ emoji_id != -1？
           │  └─ 是 → 发送表情回应
           │  └─ 否 → 不发送表情
           └─ 记录日志，返回
```


## 错误处理

如果表情发送失败（例如表情 ID 无效、网络问题等）：

1. ✅ 系统仍会正常拒绝请求
2. ⚠️ 记录错误日志
3. 🔄 不会影响后续请求处理

```python
try:
    await Fun.set_msg_emoji_like(bot, message_id, emoji_id)
except Exception as e:
    logger.error(f"发送表情失败: {e}")
    # 继续执行，不中断
```

## 配置生效

修改 `rejection_emoji_id` 后：

1. 重启 NoneBot 机器人
2. 新配置在启动时加载到全局配置对象
3. 后续被拒绝的请求会使用新配置

## 技术细节

### 实现方式

被拒绝时的处理流程：

```python
if not response_lock_manager.try_acquire(group_id):
    logger.warning(f"拒绝请求（群组 {group_id}）")

    # 检查是否需要发送表情
    if config.chat_model.rejection_emoji_id != -1:
        try:
            # 调用 Fun 模块的表情API
            await Fun.set_msg_emoji_like(
                bot=bot,
                message_id=msg_id,
                emoji_id=config.chat_model.rejection_emoji_id
            )
            logger.debug(f"表情已发送（ID {config.chat_model.rejection_emoji_id}）")
        except Exception as e:
            # 表情发送失败时记录错误，但不中断流程
            logger.error(f"表情发送失败: {e}")

    return
```

### 原子性保证

- 表情发送失败不会导致系统异常
- 拒绝请求的处理不受表情发送影响
- 锁的释放和获取不涉及表情回应

## 常见问题

**Q: 表情 ID 从哪里获取？**
A: 可以通过 QQ API 文档查询，或通过试错找到适合的表情。

**Q: 支持自定义表情吗？**
A: 不支持，只能使用 QQ 内置的表情。

**Q: 表情发送失败会怎样？**
A: 系统仍会拒绝请求，只是没有表情提示。日志中会记录错误。

**Q: 可以同时设置多个表情吗？**
A: 不可以，只能配置一个 ID。如需多个，可以在不同的配置组中配置。

**Q: 表情是否会对消息排序产生影响？**
A: 不会。表情只是一种反应，不会改变消息的顺序或状态。
