# 用户画像管理器（LongMemory/UserProfile）使用说明

本文档介绍 LongMemory 中的“用户画像管理器”如何初始化与使用，包括：写入、单用户检索、跨用户检索、反查归属、更新与删除。

> 适用模块：`plugins/GTBot/services/LongMemory/UserProfile.py`

说明：目前 `UserProfile` 已默认切换为 Qdrant 后端实现（兼容保留旧 import 路径）。底层具体实现位于：

- `plugins/GTBot/services/LongMemory/qdrant_user_profile.py`

---

## 1. 组件概览

### 1.1 核心类

- `UserProfile`：用户画像的存储与检索服务（默认 Qdrant 后端；为兼容旧代码保留该类名与导入路径）。
- `QdrantUserProfile`：用户画像的存储与检索服务（基于 Qdrant `AsyncQdrantClient`，与 `UserProfile` 提供一致的方法）。

### 1.2 依赖

- Qdrant：提供向量数据库能力（`upsert/query_points/scroll/retrieve/delete`）。
- `VectorGenerator`：向量生成器协议，负责把文本转为向量。
  - 约定：
    - 单条文本向量为 1D：`(dim,)`
    - 批量文本向量为 2D：`(n, dim)`
    - dtype：`np.float32`

---

## 2. 初始化

### 2.1 创建 UserProfile 实例

`UserProfile.create(...)` 会确保 Qdrant collection 存在（不存在则创建），并返回服务对象。

示例：

```python
from qdrant_client import AsyncQdrantClient

from plugins.GTBot.services.LongMemory.UserProfile import UserProfile
from plugins.GTBot.services.LongMemory.VectorGenerator import OpenaiVectorGenerator

client = AsyncQdrantClient(url="http://127.0.0.1:6333")
vector_generator = OpenaiVectorGenerator(...)

user_profile = await UserProfile.create(
    collection_name="user_profile",
    client=client,
    vector_generator=vector_generator,
  # 推荐显式指定向量维度，避免启动时多一次 embedding 探测
  vector_size=1024,
)
```

### 2.2 创建 QdrantUserProfile 实例（显式使用实现类）

如果你希望直接引用实现类（而不是通过兼容入口 `UserProfile`），可以使用 `QdrantUserProfile`：

示例：

```python
from qdrant_client import AsyncQdrantClient

from plugins.GTBot.services.LongMemory.VectorGenerator import OpenaiVectorGenerator
from plugins.GTBot.services.LongMemory.qdrant_user_profile import QdrantUserProfile

client = AsyncQdrantClient(url="http://127.0.0.1:6333")
vector_generator = OpenaiVectorGenerator(...)

user_profile = await QdrantUserProfile.create(
  collection_name="user_profile",
  client=client,
  vector_generator=vector_generator,
  # 推荐显式指定向量维度，避免启动时多一次 embedding 探测
  vector_size=1024,
)
```

## 3. 写入画像

### 3.1 `add_user_profile`

支持两种调用方式（二选一）：

1. 传 `user_id + 文本/文本列表`

```python
await user_profile.add_user_profile(123456, "喜欢打游戏")
await user_profile.add_user_profile(123456, ["喜欢打游戏", "讨厌香菜"])
```

2. 直接传入模型 `UserProfileModel`

```python
from plugins.GTBot.services.LongMemory.model import UserProfile

await user_profile.add_user_profile(UserProfile(id=123456, description=["喜欢打游戏"]))
```

写入行为：

- `text`：与描述文本一一对应
- `vector`：二维 `np.ndarray`，shape 为 `(n, dim)`
- `payload`：会写入 `id` 与时间戳字段（`creation_time/last_updated/last_read`）

`doc_id` 说明：

- Qdrant 版本默认使用 UUID 作为 point id（字符串形式）。

## 4. 单用户画像检索（按 user_id）

### 4.1 `get_user_profiles`

```python
profile = await user_profile.get_user_profiles(
    user_id=123456,
    limit=10,
    text=None,
    sort_by="auto",
    sort_order="desc",
)
```

返回类型：`UserProfileWithDescriptionIds`

- `id`：用户 ID
- `description`：`UserProfileDescriptionWithId` 列表，每条包含：
  - `doc_id`：数据库记录 ID（Qdrant point id，UUID 字符串）
  - `text`：描述文本

排序规则：

- `sort_by="auto"`：
  - 如果传了 `text`，则按相似度检索
  - 否则按 `last_updated` 排序
- `sort_by="text"`：使用向量检索（带 `user_id` 过滤）获取相似描述
- `sort_by in ("creation_time", "last_updated", "last_read")`：
  - 先按 `user_id` 过滤拉取候选集
  - 再在本地按时间字段排序并截断到 `limit`

## 5. 跨用户画像检索（全库向量检索）

### 5.1 `search_user_profiles`

该方法用于“跨用户”检索：在整个集合范围内查询最相似的画像描述。

#### 单条查询

```python
hits = await user_profile.search_user_profiles(
    "喜欢打游戏",
    n_results=20,
    order_by="distance",
    order="asc",
)
```

#### 批量查询

```python
hits_list = await user_profile.search_user_profiles(
    ["喜欢打游戏", "讨厌香菜"],
    n_results=10,
    order_by="similarity",
    order="desc",
)
```

返回类型：

- 单条查询：`list[UserProfileSearchHit]`
- 批量查询：`list[list[UserProfileSearchHit]]`

`UserProfileSearchHit` 字段：

- `doc_id`：数据库记录 ID
- `user_id`：所属用户 ID
- `text`：描述文本
- `distance`：距离（通常越小越相似，取决于集合度量）
- `similarity`：便捷相似度（当前实现为 `1 - distance`）

Qdrant 版本差异：

- Qdrant 返回的是 `score`（越大越相似，尤其在 COSINE 下）。
- 在 `QdrantUserProfile.search_user_profiles` 中：
  - `similarity = score`
  - `distance = 1 - score`（仅作便捷展示，不代表严格意义上的“距离”）

排序：

- `order_by="distance"`：按 `distance` 排序
- `order_by="similarity"`：按 `similarity` 排序
- `order="asc"|"desc"`：升/降序

## 6. 通过 doc_id 反查归属

### 6.1 `get_owner_by_doc_id`

返回用户 ID：

```python
owner = await user_profile.get_owner_by_doc_id("<doc_id>", return_type="user_id")
owners = await user_profile.get_owner_by_doc_id(["<doc1>", "<doc2>"], return_type="user_id")
```

返回所属用户的画像：

```python
profile = await user_profile.get_owner_by_doc_id("<doc_id>", return_type="profile")
profiles = await user_profile.get_owner_by_doc_id(["<doc1>", "<doc2>"], return_type="profile")
```

## 7. 更新数据（按 doc_id）

### 7.1 `update_by_doc_id`

目标：允许传入数据库 ID 或 ID 列表，以及其他可选字段（`descriptions/user_id/xxx_time`），只更新传入字段，其他保持不变；`doc_id` 不允许更新。

#### 单条更新

```python
updated = await user_profile.update_by_doc_id(
    "<doc_id>",
    descriptions="新的描述文本",
    last_updated=time.time(),
)
```

#### 批量更新（标量广播）

```python
updated = await user_profile.update_by_doc_id(
    ["<doc1>", "<doc2>"],
    user_id=123456,
    last_read=time.time(),
)
```

#### 批量更新（列表一一对应）

```python
updated = await user_profile.update_by_doc_id(
    ["<doc1>", "<doc2>"],
    descriptions=["新文本1", "新文本2"],
    user_id=[111, 222],
)
```

更新规则：

- 只要 `descriptions` 被更新：
  - 会更新对应记录的文本字段（`text`）与 payload 中的时间字段
  - 并重新计算向量后写回向量数据库
- 如果只更新元数据字段（`user_id/creation_time/last_updated/last_read`）：
  - 不会改变文本与向量

异常：

- 列表长度与 `doc_id` 不一致会抛 `ValueError`
- 传入的 `doc_id` 有不存在的会抛 `ValueError`

## 8. 删除

### 8.1 按 doc_id 删除

```python
deleted = await user_profile.delete_by_doc_id("<doc_id>")
deleted = await user_profile.delete_by_doc_id(["<doc1>", "<doc2>"])
```

### 8.2 按 user_id 删除该用户全部数据

```python
deleted = await user_profile.delete_all_by_user_id(123456, page_size=1000)
```

实现说明：会分页拉取该用户的所有 ids，再分批 delete，避免一次性请求过大。

---

## 9. 常见建议

- 距离/相似度：不同集合度量下 `distance` 含义可能不同，若要严格相似度请按你的度量进行换算。
- 更新时间戳：如果你希望“更新 descriptions 时自动刷新 last_updated”，可以在调用 `update_by_doc_id` 时显式传入 `last_updated=time.time()`。
- 批量操作：尽量用列表批量传入，减少网络往返与 Python 开销。

---

## 10. 相关数据结构（快速索引）

- `UserProfileModel`：写入用模型（`id: int`, `description: list[str]`）
- `UserProfileWithDescriptionIds`：单用户画像返回（每条描述带 `doc_id`）
- `UserProfileDescriptionWithId`：`doc_id + text`
- `UserProfileSearchHit`：跨用户检索命中项（含 `doc_id/user_id/text/distance/similarity`）
